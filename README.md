# Linux Virus

Linux Virus は、macOS に常駐して Linux コマンドのクイズを出題するデモ用アプリです。

画面右上に小さなウィンドウとして常駐し、クリップボードの更新を検知すると問題を表示します。コピーした文字列の先頭が対応済みの Linux コマンドだった場合は、そのコマンドに関する問題を優先して出題します。

このブランチはデモ体験用にフロントエンドだけを残しています。問題取得・ログイン・回答記録などの API はリモートのデモ用バックエンドを利用します。

## 使い方

### セットアップ

`uv` が未インストールの場合は先にインストールします。

```bash
brew install uv
```

依存関係を同期します。

```bash
uv sync
```

### 起動

リポジトリルートで以下を実行します。

```bash
uv run linux-virus-frontend
```

起動すると画面右上に常駐ウィンドウが表示されます。

### 体験手順

1. 必要に応じてユーザーボタンからログインします。
2. 任意のテキストをコピーすると、通常問題が出題されます。
3. コピー内容の先頭単語が対応済み Linux コマンドの場合は、そのコマンドに関する問題が優先されます。
4. 問題画面では、トークンの並び替えまたはターミナル入力欄への入力で回答します。
5. 正解後は解説を確認し、`閉じる` で常駐画面に戻ります。

### 終了

ターミナルで `Ctrl+C` を押すか、アプリのグローバルショートカット `Ctrl+Option+Q` で終了します。

ターミナルに戻れない場合は、別のターミナルから以下を実行します。

```bash
pkill -f linux-virus-frontend
```

## macOS 権限

グローバルショートカット、入力制限、クリップボード監視には macOS の権限が必要になる場合があります。

1. 初回実行時に許可ダイアログが出たら許可します。
2. 出ない場合は `システム設定 > プライバシーとセキュリティ` から、実行元のターミナルアプリに以下を許可します。
   - アクセシビリティ
   - 入力監視
3. 許可後は起動中のターミナルアプリを完全に終了し、再起動してから Linux Virus を起動します。

## 設定

デフォルトの API 接続先は `frontend/.env.public` の `API_BASE_URL` に定義されています。

```text
API_BASE_URL=https://linux-virus-backend.onrender.com
```

別の API を使いたい場合は、起動時に環境変数で上書きできます。

```bash
API_BASE_URL=http://127.0.0.1:8000 uv run linux-virus-frontend
```

## 機能一覧

- macOS 常駐アプリとして動作し、画面右上に最小化ウィンドウを表示します。
- 通常問題はクリップボードの更新をトリガーに出題されます。
- クリップボード監視は短い間隔のポーリングで行います。`Command+C` だけでなく、右クリックメニューなどによるコピーも検知対象です。
- コピー内容が文字列で、先頭単語が `.env.public` の `SUPPORTED_LINUX_COMMANDS` に含まれる場合はコマンド別問題を取得します。
- コマンド別問題がない場合や、先頭単語が対応済み Linux コマンドではない場合は通常の出題ロジックにフォールバックします。
- 設定画面では休止時間とパーソナライズ出題の有効/無効を変更できます。休止時間 `0` は休止なしです。
- 休止中は通常問題のクリップボード発火を抑止し、最小化画面に青い残り時間タイマーを表示します。
- ログイン中は回答ログ、レーティング、連続正解数、パーソナライズ出題を利用できます。
- 通常問題で誤答するとウイルスカウントが増え、ウイルス問題の出題対象になります。
- ウイルス問題は通常問題とは独立したウィンドウで出題されます。正解するとウイルスカウントが減ります。
- ワクチンはウイルス問題をスキップする機能です。1日3個まで使用でき、毎朝 4:00 にリセットされます。
- キーボード監視はショートカット用途に限定しています。通常問題の出題トリガーには使いません。
- ショートカットは `Ctrl+Option+V`（ウイルス問題を手動表示）、`Ctrl+Option+Q`（終了）、`Ctrl+Option+F`（ワクチン回復・デバッグ用）です。
