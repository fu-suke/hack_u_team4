# Linux Virus

Linux Virus は、macOS に常駐して特定のタイミングで Linux コマンドのクイズを表示する学習支援アプリです。

作業中にクリップボードの更新を検知すると問題を出し、表示されたトークンの並び替えやターミナル入力で Linux コマンドを完成させます。

コピーされた文字列の先頭が対応済みの Linux コマンドだった場合は、そのコマンドに関する問題を優先して出題します。該当する問題がない場合や Linux コマンドではない場合は、通常の出題ロジックにフォールバックします。

正解時には解説を確認でき、ログインして使うと回答履歴に応じたパーソナライズ問題が出題されます。通常問題で間違えた問題はウイルス問題として蓄積され、一定間隔で独立ウィンドウに出題されます。

## 使い方

1. アプリを起動すると、画面右上に小さな常駐ウィンドウが表示されます。
2. 必要に応じてユーザーボタンからログインします。ログイン中は回答履歴に応じた問題が出題されます。
3. 設定ボタンから休止時間を分単位で設定できます。`0` は休止なしです。
4. テキストをコピーすると、クリップボードの更新を検知して通常問題が出題されます。`Command+C` だけでなく、右クリックメニューなどによるコピーも対象です。
5. コピー内容が文字列で、半角スペースで区切った先頭単語が対応済み Linux コマンドの場合は、そのコマンドに関する問題を優先して出題します。
6. 問題画面では、表示されたトークンを並び替えるか、ターミナル入力欄にコマンドを入力して回答します。
7. 正解すると解説とサンプル出力を確認できます。不正解の場合は正解するまで再挑戦できます。
8. 通常問題で誤答すると、その問題のウイルスカウントが増えます。ウイルス問題は一定間隔で独立ウィンドウとして出題され、正解するとウイルスカウントが減ります。
9. ウイルス問題はワクチンでスキップできます。ワクチンは1日3個まで使用できます。
10. 主なショートカットは `Ctrl+Option+V`（ウイルス問題を手動表示）、`Ctrl+Option+Q`（終了）、`Ctrl+Option+F`（ワクチン回復・デバッグ用）です。

## macOS 権限

グローバルホットキーとバックグラウンドのキーボード入力監視には macOS の権限が必要です。

1. 初回実行時に許可ダイアログが出たら許可します。
2. 出ない場合は `システム設定 > プライバシーとセキュリティ` で以下を許可します。
   - アクセシビリティ
   - 入力監視
3. ターミナルから起動している場合、許可対象は Linux Virus ではなく実行元です。
   - ターミナル
   - iTerm
   - VS Code
   - Cursor
4. 許可後に起動中のターミナルアプリを完全に終了し、再起動してから Linux Virus を起動します。

## 実行

通常利用では、以下のコマンドで起動できます。

### uv のインストール

`uv` が未インストールの場合は、先にインストールします。

```bash
brew install uv
```

### アプリ起動

リポジトリルートで以下を実行します。

```bash
API_BASE_URL=https://linux-virus-backend.onrender.com uv run --package linux-virus-frontend linux-virus-frontend
```

## 終了

ターミナルで `Ctrl+C` を押して終了できます。

ターミナルにフォーカスが戻せない場合は、別のターミナルから以下でプロセスを終了できます。

```bash
pkill -f "resident_poc"
# または honcho ごと終了
pkill -f honcho
```

ポート 8000 が使用中（`Address already in use`）で起動できない場合は、残っているプロセスを終了します。

```bash
lsof -ti :8000 | xargs kill -9
```

## 環境構成と起動パターン

バックエンドと DB は、ローカルでもリモート（Render）でも動作します。接続先は環境変数で切り替えます。

- `DATABASE_URL`: バックエンドが接続する DB。未指定ならローカルの SQLite（`backend/app.db`）を使用
- `API_BASE_URL`: フロントが接続するバックエンド。未指定なら `.env.public` の値に従う

切り替えは、起動時に環境変数を指定することで行います（環境変数は `.env.public` の値より優先されます）。

### パターン1: バックエンドもDBもローカル

バックエンド・DB ともにローカル。オフライン開発や、共有 DB を汚したくない場合に使用。

初回またはデータを初期状態に戻したい場合は、DB の初期データを投入します。

```bash
uv sync --all-packages
uv run --directory backend python seed.py --force
```

アプリを起動する場合は、リポジトリルートで以下を実行します。

```bash
uv run honcho start
```

### パターン2: バックエンドはローカル、DB はリモート

バックエンドのコードを触りながら、共有の Render DB の状態を見たい場合に使用。

```bash
DATABASE_URL="<Render の External Database URL>" uv run --directory backend uvicorn app.main:app --reload
```

### パターン3: すべてリモート
 
バックエンドは Render 上で常時稼働中。フロント開発や本番動作確認に使用。フロント起動時に Render のバックエンドを向ける。
 
`.env.public` のデフォルトはローカルバックエンド（`http://127.0.0.1:8000`）になっているため、Render に接続したい場合は起動時に環境変数で上書きする。
 
```bash
API_BASE_URL=https://linux-virus-backend.onrender.com uv run --package linux-virus-frontend linux-virus-frontend
```
 
`.env.public` は編集せず、接続先の切り替えは常に起動時の環境変数で行う（環境変数は `.env.public` の値より優先される）。
 
**Render の External Database URL は 秘匿情報** です。<u>公開の場（パブリックな Discord チャンネルなど）には貼らないでください。</u>

## Render（バックエンド / DB のホスティング）

バックエンドと DB は Render にホストされています。

- バックエンド URL: `https://linux-virus-backend.onrender.com`
- ヘルスチェック: `https://linux-virus-backend.onrender.com/health`
- インフラ構成：`render.yaml`

### 自動デプロイ

`master` ブランチへの push で、Render に自動デプロイされます。手動操作は不要です。

### Render の DB にシードを投入する

ローカルから External Database URL を指定してシードを実行します。

```bash
DATABASE_URL="<Render の External Database URL>" uv run --directory backend python seed.py
```

データを初期状態に戻して再投入したい場合は `--force` を付けます。

```bash
DATABASE_URL="<Render の External Database URL>" uv run --directory backend python seed.py --force
```

> Render の無料プランはアクセスがない間スリープします。スリープからの復帰には最初のリクエストで数秒〜十数秒かかります。デモ前には一度アクセスして起こしておくと安心です。

## 機能一覧

- macOS 常駐アプリとして動作し、画面右上に最小化ウィンドウを表示します。
- 通常問題はクリップボードの更新だけをトリガーに出題されます。通常問題用のタイマーや任意キー監視はありません。
- クリップボード監視は短い間隔のポーリングで行います。右クリックメニューやアプリ内メニューによるコピーも、クリップボードが更新されれば検知対象です。
- コピー内容が文字列で、先頭単語が `.env.public` の `SUPPORTED_LINUX_COMMANDS` に含まれる場合は `/questions/by-command` でコマンド別問題を取得します。
- コマンド別問題が取得できない場合、または先頭単語が対応済み Linux コマンドではない場合は、ログイン状態と設定に応じてパーソナライズ問題またはランダム問題を出題します。
- 設定画面では休止時間とパーソナライズ出題の有効/無効を変更できます。休止時間 `0` は休止なしです。
- 休止中は通常問題のクリップボード発火を抑止し、最小化画面に青い残り時間タイマーを表示します。
- ログインはユーザー名と4桁 PIN で行います。ログイン中は回答ログ、レーティング、連続正解数、パーソナライズ出題を利用できます。
- 問題はトークンの並び替え、またはターミナル入力欄への入力で回答できます。
- 通常問題で誤答するとウイルスカウントが増え、ウイルス問題の出題対象になります。
- ウイルス問題は通常問題とは独立したウィンドウで出題されます。正解するとウイルスカウントが減ります。
- ワクチンはウイルス問題をスキップする機能です。1日3個まで使用でき、毎朝 4:00 にリセットされます。
- キーボード監視はショートカット用途に限定しています。通常問題の出題トリガーには使いません。
- ショートカットは `Ctrl+Option+V`（ウイルス問題を手動表示）、`Ctrl+Option+Q`（終了）、`Ctrl+Option+F`（ワクチン回復・デバッグ用）です。
- グローバルショートカット、入力制限、クリップボード監視には macOS のアクセシビリティ権限や入力監視権限が必要になる場合があります。
- バックエンドは FastAPI、DB は SQLite または Render 上の DB を使用します。接続先は `API_BASE_URL` と `DATABASE_URL` で切り替えます。
