# 作問ポリシー
 
 
## 1. 全体方針
 
- **対象**: Linuxコマンドを覚えられない初心者〜中級者
- **形式**: 並べ替えクイズ(コマンド要素を正しい順に並べる)
- **環境**: GNU/Linux標準で動作するもののみ(macOS/BSD固有は除外)
---
 
## 2. 難易度
 
3段階(Lv1〜3)で設計する。
応用・上級は今回のスコープ外。
 
| Lv | 名称 | 要素数 | ダミー数 | 構造 |
|---:|---|---|---|---|
| 1 | 入門 | 2〜3 | 0〜1 | 単一コマンド |
| 2 | 基本 | 3〜5 | 1〜2 | 単一コマンド + オプション/引数 |
| 3 | 標準 | 4〜7 | 2〜3 | パイプ1段、リダイレクト |
| 4 | 応用 | 6〜8 | 3〜4 | パイプ2段以上、サブシェル、プロセス置換 |
| 5 | 上級 | 8〜10 | 4〜5 | 複雑な構造(パイプ複数段、サブシェルの入れ子など) |
 
 
---
 
## 3. 選択肢の粒度
 
**ルール: スペース区切りで分割。記号も独立した選択肢にする。**
 
| コマンド | 選択肢 |
|---|---|
| `ls -la /etc` | `ls`, `-la`, `/etc` |
| `ls -l -a /etc` | `ls`, `-l`, `-a`, `/etc` |
| `cat file.txt \| grep error` | `cat`, `file.txt`, `\|`, `grep`, `error` |
| `find . -name "*.log"` | `find`, `.`, `-name`, `"*.log"` |
 
---
 
## 4. 別解の扱い
 
別解には2つのタイプがあり、それぞれのタイプでポリシーが異なる。
 
- **タイプA**: 同じ選択肢の集合を別の順序で並べる別解(例: `ls -l -a` と `ls -a -l`)
- **タイプB**: 使う選択肢の集合自体が違う別解(例: `cat file | grep error` と `grep error file`)

ポリシー:
 
- タイプA別解は**許容**し、 **全パターンを `answers` テーブルに登録**する
- 任意順オプションは **3個まで**(別解6通りが上限。4個以上=24通りはNG)
  - 例: `ls -l -a -h`(3個任意順 → `3! = 6通り` を登録、OK)
  - 例: `ls -l -a -h -S`(4個任意順 → `4! = 24通り` になりNG)
- タイプB別解が成立する題材は **出題しない**
  - 技術的には登録・採点可能だが、以下の理由で推奨しない:
    - **学習意図がぼやける**: UUOC(`cat file | grep` のような冗長な書き方)を「OKな書き方」として教えてしまう恐れがある
    - **ダミーを置く余地が減る**: 別解で使う要素が増えるため、紛らわしいダミーを追加する余地が減り、難易度設計が窮屈になる
---
 
## 5. 問題文の書き方
 
ルール:
 
- 動作・要件が一意に決まる書き方をする(タイプB別解を生まないため)
- 対象ファイル名・パス等を具体的に明示する
### 良い例
 
- ファイル `app.log` から "error" を含む行を表示する
- カレントディレクトリのファイル一覧を、<u>隠しファイルも含めて</u>詳細表示する
- ディレクトリ `/var/log` 配下から拡張子 `.log` のファイルを<u>再帰的</u>に検索する
- プロセス一覧から `nginx` を含む行だけを表示する
### 悪い例
 
- ファイルから "error" を含む行を見つける
  - → 対象ファイルが不明
- ファイル一覧を見やすく表示する
  - → 「見やすく」が曖昧
- カレントディレクトリのファイル一覧を表示する
  - → 隠しファイルを含むかどうかが不明。選択肢に `-a` があると判断できない
- ディレクトリから拡張子 `.log` のファイルを検索する
  - → 再帰的かどうかが不明。選択肢に `-r` があると判断できない

---
 
## 6. ダミーの作り方
 
### 紛らわしさのパターン(難易度別の推奨)
 
| パターン | 例(正解→ダミー) | Lv1 | Lv2 | Lv3 |
|---|---|:-:|:-:|:-:|
| ①紛らわしいオプション/引数(似ているもの、不要なもの) | `-r` → `-R`, `-i` / `ls -l` に不要な `-a` を混ぜる | ✓ | ✓ | ✓ |
| ②引数の値違い | `chmod 755` → `644` | ✓ | ✓ | ✓ |
| ③似た役割の別コマンド | `cat` → `less`, `more` | ✓ | ✓ | ✓ |
| ④クォートの違い | `"*.log"` → `*.log`(クォート無しで展開されてしまう) | | | ✓ |
 
### やってはいけないダミー
 
- ❌ 完全に無関係な要素(`grep` の問題に `vim`)
- ❌ タイプB別解を生む要素(例: `ls` の問題に `.` を混ぜると `ls .` という別解が成立してしまう)
---
 
## 7. カテゴリ
 
**コマンド単位で分類**(`ls`, `grep`, `find`, `tar` など)。
 
パイプを含む問題は、主要コマンド(問題のメインとなるコマンド)で分類する。
 
将来的に「ファイル操作」「テキスト処理」「プロセス管理」などのカテゴリも設ける可能性はあるが、現状はコマンド単位で十分と判断

---
 
## 付録1:サンプル問題
 
### Lv1(入門)
 
**問題文**: カレントディレクトリのパスを表示する
 
**選択肢**: `pwd`, `cd`(ダミー)
 
**正解**: `pwd`
 
**カテゴリ**: pwd / **難易度**: 1
 
---
 
### Lv2(基本)
 
**問題文**: ファイル `app.log` から "error" を含む行を表示する
 
**選択肢**: `grep`, `"error"`, `app.log`, `-v`(ダミー), `cat`(ダミー)
 
**正解**: `grep "error" app.log`
 
**カテゴリ**: grep / **難易度**: 2
 
---
 
### Lv3(標準)
 
**問題文**: カレントディレクトリにあるファイルの中で、サイズが大きい順に上位5件のファイル名を表示する
 
**選択肢**: `ls`, `-l`, `-S`, `|`, `head`, `-n`, `5`, `awk`(ダミー)
 
**正解**(2通り、`-l` と `-S` が任意順):
- `ls -l -S | head -n 5`
- `ls -S -l | head -n 5`
**カテゴリ**: ls / **難易度**: 3
---

## 付録2: 作問後のセルフチェック
 
問題を確定する前に、以下を確認:
 
### 題材
- [ ] GNU/Linux標準で動作する
- [ ] タイプB別解が成立しない
- [ ] 任意順オプションが3個以下
### 問題文
- [ ] 動作・要件が一意に決まる
- [ ] 対象ファイル名・パス等が明示されている
### 選択肢
- [ ] スペース区切りで分割されている
- [ ] パイプ・リダイレクト記号が独立した選択肢になっている
- [ ] 正解要素が7個以下、合計10個以下
### 正解
- [ ] タイプA別解の全パターンを登録した
### ダミー
- [ ] 難易度別の推奨パターンに沿っている
- [ ] アンチパターンに該当しない
### 難易度
- [ ] 設定難易度と内容(要素数・ダミー数・構造)が整合している
