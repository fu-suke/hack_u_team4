# 作問ポリシー


## 1. 全体方針

- **対象**: Linuxコマンドを覚えられない初心者〜中級者
- **形式**: 並べ替えクイズ(コマンド要素を正しい順に並べる)
- **環境**: GNU/Linux標準で動作するもののみ(macOS/BSD固有は除外)

---

## 2. 難易度

3段階(Lv1〜3)で設計する。
応用・上級は今回のスコープ外。

| Lv | 名称 | 要素数 | ダミー数 | 構造 |
|---:|---|---|---|---|
| 1 | 入門 | 2〜3 | 0〜1 | 単一コマンド |
| 2 | 基本 | 3〜5 | 1〜2 | 単一コマンド + オプション/引数 |
| 3 | 標準 | 4〜7 | 2〜3 | パイプ1段、リダイレクト |
| 4 | 応用 | 6〜8 | 3〜4 | パイプ2段以上、サブシェル、プロセス置換 |
| 5 | 上級 | 8〜10 | 4〜5 | 複雑な構造(パイプ複数段、サブシェルの入れ子など) |


---

## 3. 選択肢の粒度

**ルール: スペース区切りで分割。記号も独立した選択肢にする。**

| コマンド | 選択肢 |
|---|---|
| `ls -la /etc` | `ls`, `-la`, `/etc` |
| `ls -l -a /etc` | `ls`, `-l`, `-a`, `/etc` |
| `cat file.txt \| grep error` | `cat`, `file.txt`, `\|`, `grep`, `error` |
| `find . -name "*.log"` | `find`, `.`, `-name`, `"*.log"` |

### 合成形オプションの扱い

頻出する組み合わせは**合成形で1つの選択肢**として出題することを推奨。

- 並び替え要素数を抑制できる
- 実務での書き方に近い
- タイプA別解の登録爆発を防げる

例:

- `tar -czf archive.tar.gz data/`(`-c`, `-z`, `-f` を個別ではなく `-czf` として出題)
- `ps aux`(`a`, `u`, `x` を分割しない)
- `du -sh /var/log`(`-s -h` ではなく `-sh`)
- `ls -la`(分割形の `-l -a` も可。意図に応じて使い分け)
- `rm -rf dir/`(`-r -f` ではなく `-rf`)

分割形(`-l -a` 等)で出題するのは「オプションを個別に学ばせたいLv1〜Lv2の単純な問題」のみとする。

---

## 4. 別解の扱い

別解には2つのタイプがあり、それぞれのタイプでポリシーが異なる。

- **タイプA**: 同じ選択肢の集合を別の順序で並べる別解(例: `ls -l -a` と `ls -a -l`)
- **タイプB**: 使う選択肢の集合自体が違う別解(例: `cat file | grep error` と `grep error file`)

ポリシー:

- タイプA別解は**許容**し、**全パターンを `answers` テーブルに登録**する
- 任意順オプションは **3個まで**(別解6通りが上限。4個以上=24通りはNG)
  - 例: `ls -l -a -h`(3個任意順 → `3! = 6通り` を登録、OK)
  - 例: `ls -l -a -h -S`(4個任意順 → `4! = 24通り` になりNG)
- タイプB別解が成立する題材は **出題しない**
  - 技術的には登録・採点可能だが、以下の理由で推奨しない:
    - **学習意図がぼやける**: UUOC(`cat file | grep` のような冗長な書き方)を「OKな書き方」として教えてしまう恐れがある
    - **ダミーを置く余地が減る**: 別解で使う要素が増えるため、紛らわしいダミーを追加する余地が減り、難易度設計が窮屈になる

---

## 5. 問題文の書き方

ルール:

- 動作・要件が一意に決まる書き方をする(タイプB別解を生まないため)
- 対象ファイル名・パス等を具体的に明示する

### 良い例

- ファイル `app.log` から "error" を含む行を表示する
- カレントディレクトリのファイル一覧を、<u>隠しファイルも含めて</u>詳細表示する
- ディレクトリ `/var/log` 配下から拡張子 `.log` のファイルを<u>再帰的</u>に検索する
- プロセス一覧から `nginx` を含む行だけを表示する

### 悪い例

- ファイルから "error" を含む行を見つける
  - → 対象ファイルが不明
- ファイル一覧を見やすく表示する
  - → 「見やすく」が曖昧
- カレントディレクトリのファイル一覧を表示する
  - → 隠しファイルを含むかどうかが不明。選択肢に `-a` があると判断できない
- ディレクトリから拡張子 `.log` のファイルを検索する
  - → 再帰的かどうかが不明。選択肢に `-r` があると判断できない

---

## 6. ダミーの作り方

### 紛らわしさのパターン(難易度別の推奨)

| パターン | 例(正解→ダミー) | Lv1 | Lv2 | Lv3 |
|---|---|:-:|:-:|:-:|
| ①紛らわしいオプション/引数(似ているもの、不要なもの) | `-r` → `-R`, `-i` / `ls -l` に不要な `-a` を混ぜる | ✓ | ✓ | ✓ |
| ②引数の値違い | `chmod 755` → `644` | ✓ | ✓ | ✓ |
| ③似た役割の別コマンド | `cat` → `less`, `more` | ✓ | ✓ | ✓ |
| ④クォートの違い | `"*.log"` → `*.log`(クォート無しで展開されてしまう) | | | ✓ |
| ⑤合成形オプションの取り違え | `-czf` → `-xzf`(動作が逆) | | ✓ | ✓ |

### やってはいけないダミー

- ❌ 完全に無関係な要素(`grep` の問題に `vim`)
- ❌ タイプB別解を生む要素(例: `ls` の問題に `.` を混ぜると `ls .` という別解が成立してしまう)

## 7. カテゴリ

**コマンド単位で分類**(`ls`, `grep`, `find`, `tar` など)。

パイプを含む問題は、主要コマンド(問題のメインとなるコマンド)で分類する。

将来的に「ファイル操作」「テキスト処理」「プロセス管理」などのカテゴリも設ける可能性はあるが、現状はコマンド単位で十分と判断。

## 8. 出題範囲

### リダイレクト・パイプ記号

| 記号 | 用途 | 対象難易度 |
|---|---|---|
| `\|` | パイプ | Lv3 |
| `>` | 上書きリダイレクト | Lv2〜Lv3 |
| `>>` | 追記リダイレクト | Lv3 |
| `2>` | エラーリダイレクト | Lv3(応用寄り) |

### 引数の使い方

- 引数には**具体的なファイル名・ディレクトリ名**を入れて構わない
  - 例: `cp src/ dst/`、`mv file.txt /tmp/`、`grep "error" app.log > result.txt`
- 引数を含めることで問題のバリエーションが増える
- 引数は問題文と整合させる(ポリシー5「対象ファイル名を明示」と一致)

### スコープ外のコマンド・オプション

以下は意図的に出題範囲から除外する。

- `ssh`: 並べ替え問題として要素が少なく、学習価値が低い
- `sed`, `awk`: 初心者〜中級者(Lv1〜3)のスコープを超える
- `tar` の bzip2 系オプション(`-cjf`, `-xjf`): マニアックすぎる。gzip版に絞る
- `curl` の `-X`, `-H`, `-d`: REST API系で中級者以上向け

## 9. データスキーマ

問題は以下のスキーマで管理する。

| カラム | 型 | 内容 |
|---|---|---|
| `difficulty` | 整数 | 難易度(1〜3) |
| `category` | 配列(文字列) | カテゴリ(コマンド名)。配列形式なので将来的に複数カテゴリも可能 |
| `prompt` | 文字列 | 問題文 |
| `choices` | 配列(文字列) | 選択肢のリスト |
| `answers` | 配列の配列(整数) | 正解パターン。各内側配列が1つの正解、要素は`choices`の**1-indexed**インデックス |
| `tutorial` | 文字列 | 正解後の「ひとこと解説」+ 紛らわしいダミーの説明 |

### サンプル

| difficulty | category | prompt | choices | answers | tutorial |
|---|---|---|---|---|---|
| 1 | `["pwd"]` | カレントディレクトリの絶対パスを表示する。 | `["pwd", "cd"]` | `[[1]]` | (記入例は付録1参照) |
| 2 | `["ls"]` | カレントディレクトリのファイル一覧を、隠しファイルも含めて詳細表示する。 | `["ls", "-l", "-a", "-h", "/tmp"]` | `[[1, 2, 3], [1, 3, 2]]` | (記入例は付録1参照) |

### `answers` の書き方

- インデックスは **1始まり**
- タイプA別解(同じ集合の順序違い)は **全パターンを登録** する
- 例: `["ls", "-l", "-a", "-h", "/tmp"]` で `ls -l -a` と `ls -a -l` の両方が正解
  - → `answers: [[1, 2, 3], [1, 3, 2]]`
  - `-h` と `/tmp` はダミーなので `answers` には含まれない

---

## 10. tutorial の書き方

正解後に表示される「ひとこと解説」。以下のルールで書く。

### 含める内容

1. **コマンド・オプションの動作の解説**(なぜこれで要件を満たせるか)
2. **本質的に紛らわしいダミーの説明**(似ているが今回は不要だった理由)

### 解説するダミー / 解説しないダミー

判断基準は「学習者が**本当に迷いそうなもの**かどうか」。

| ダミーの種類 | 解説する? |
|---|---|
| 似たオプション(`-l` の問題に `-a`, `-h`) | ✅ する |
| 似たコマンド(`cat` の問題に `less`, `more`) | ✅ する |
| 引数の値違い(`755` の問題に `644`) | ✅ する |
| 合成形の取り違え(`-czf` の問題に `-xzf`) | ✅ する |
| クォートの違い(`"*.log"` の問題に `*.log`) | ✅ する |
| 無関係な引数(カレント対象の問題に `/tmp`) | ❌ しない |
| 完全に無関係なコマンド(`grep` の問題に `vim`)※そもそも出題しない | ❌ しない |

### 分量

- 全体で **2〜3文程度**を目安にする
- 長すぎる解説は読まれない。ポイントだけ簡潔に

## 付録1:サンプル問題

### Lv1(入門)

**問題文**: カレントディレクトリのパスを表示する

**選択肢**: `pwd`, `cd`(ダミー)

**正解**: `pwd`

**tutorial**: `pwd` は print working directory の略で、現在の作業ディレクトリの絶対パスを表示する。`cd` はディレクトリを移動するためのコマンド。

**カテゴリ**: pwd / **難易度**: 1

---

### Lv2(基本)

**問題文**: ファイル `app.log` から "error" を含む行を表示する

**選択肢**: `grep`, `"error"`, `app.log`, `-v`(ダミー), `cat`(ダミー)

**正解**: `grep "error" app.log`

**tutorial**: `grep` はファイルからパターンに一致する行を抽出するコマンド。`-v` は「一致**しない**行」を抽出するオプションで、今回の要件とは逆。`cat` はファイル全体を表示するコマンドで、絞り込みには使えない。

**カテゴリ**: grep / **難易度**: 2

---

### Lv3(標準)

**問題文**: カレントディレクトリにあるファイルの中で、サイズが大きい順に上位5件のファイル名を表示する

**選択肢**: `ls`, `-l`, `-S`, `|`, `head`, `-n`, `5`, `awk`(ダミー)

**正解**(2通り、`-l` と `-S` が任意順):
- `ls -l -S | head -n 5`
- `ls -S -l | head -n 5`

**tutorial**: `ls -S` でサイズの大きい順に並び、`-l` で詳細表示。`head -n 5` で先頭5件に絞る。`awk` はテキスト処理用の強力なコマンドだが、上位5件を取るだけなら `head` で十分。

**カテゴリ**: ls / **難易度**: 3

---

## 付録2: 作問後のセルフチェック

問題を確定する前に、以下を確認:

### 題材
- [ ] GNU/Linux標準で動作する
- [ ] 出題範囲(セクション8)内のコマンド・オプションを使っている
- [ ] タイプB別解が成立しない
- [ ] 任意順オプションが3個以下

### 問題文
- [ ] 動作・要件が一意に決まる
- [ ] 対象ファイル名・パス等が明示されている

### 選択肢
- [ ] スペース区切りで分割されている
- [ ] パイプ・リダイレクト記号が独立した選択肢になっている
- [ ] 合成形オプションは合成形のまま1要素として扱っている(該当する場合)
- [ ] 正解要素が7個以下、合計10個以下

### 正解
- [ ] タイプA別解の全パターンを登録した

### ダミー
- [ ] 難易度別の推奨パターンに沿っている
- [ ] アンチパターンに該当しない

### tutorial
- [ ] コマンド・オプションの動作が解説されている
- [ ] 本質的に紛らわしいダミーは解説されている
- [ ] 無関係な引数・コマンドへの解説は書いていない
- [ ] 2〜3文程度に収まっている

### 難易度
- [ ] 設定難易度と内容(要素数・ダミー数・構造)が整合している
